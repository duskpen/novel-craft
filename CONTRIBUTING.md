# 贡献指南

欢迎补充方法、模板和工具化方向，但请先遵守下面几条。

## 贡献范围

优先欢迎：

- 方法论文档优化
- 通用模板补充
- 工作流说明完善
- 不泄露私有项目的脱敏示例
- 第二阶段守门员工具的设计提案

暂不优先：

- 把仓库改写成“全自动写书平台”
- 提交未脱敏的私人小说正文
- 直接塞入大量专案设定答案

## 文档贡献原则

1. 优先写“可复用方法”，不要写“某一部小说的答案”。
2. 如果示例带有专名、世界观、剧情答案，先脱敏再提交。
3. 新增模板时，尽量保持“空骨架 + 简短说明 + 极小示例”。
4. 新增规则时，先说明它解决什么问题，再说明副作用。

## Pull Request 检查

提交前请自查：

- 有没有泄露真实项目设定、章节、进度
- 有没有把模板写成某一类角色的单一答案
- 有没有让仓库定位从“质量框架”滑向“全自动产品宣传”
- 新增内容是否和现有模块重复

## 风格

- 默认使用简体中文
- 保持表达直接、克制、可执行
- 少写口号，多写方法和边界
