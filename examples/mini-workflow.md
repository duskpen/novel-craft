# 极小示例

这个示例只演示“怎么用”，不演示完整项目。

## 1. 角色调色盘

```markdown
角色：林砚
一句话定位：很会看人脸色的底层少年

主色调：圆滑
底色：怕失去来之不易的位置
对冲 / 点缀：真被逼急时会突然不让

衍生：
- 平时先接话，不先翻脸
- 面对真正压人的秩序时，会突然安静
- 一旦有人替他挡事，他反而容易嘴变少
```

## 2. 行为禁区

```markdown
常态下绝不会做的事：
- 不会主动承认自己在意谁
- 不会在众人面前示弱
- 不会无缘无故往死路上冲

例外条件：
- 只有在别人替他担下代价时，才可能破
```

## 3. 本章输入包

```markdown
章号：12
视角：林砚
时间地点：夜里，后院井边
上一章残留：他刚知道自己不是唯一被利用的人

本章存在理由：
- 把“他会不会继续装糊涂”往前推一步

本章主轴：
- 关系试探

本章要推进的两个轴：
- 林砚和高位角色之间的信任
- 他对局势的判断方式
```

## 4. 批次记录

```markdown
## 第四批（第10-12章）

进度：从低烈度试探推进到第一次小摊牌

已用手法：
- 章尾有一次决心收尾
- 高位角色第一次没有直接压人

下一批提醒：
- 不要再用“他嘴上敷衍，她冷冷看着”这一套原样复用
- 下一批要让配角有自己的手上动作，不只负责发糖
```

## 5. 这个示例想说明什么

- 角色先立行为逻辑，再写剧情
- 每章只激活当前相关信息
- 每批都记录已经用过的写法
- “反模板”和“活人感”不是写完才补，是从输入阶段就要开始控制
