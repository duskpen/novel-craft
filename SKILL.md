# Novel-Craft

这是整个框架的核心入口。读完这份文件就能开始写。后面链接的详细文件是加深，不是前置条件。

---

## 第零步：浸泡目标风格（强制前置，不可跳过）

**任何规则都比不上"写之前先读几段你想要的风格"。**

AI 的语感来自上下文。你在对话里塞了什么样的文字，它就会输出什么样的文字。这比所有规则加起来都有效。

1. 找到目标风格——某部小说的原文、某个作者的段落、甚至作者自己满意的旧章节。
2. 写之前，把 2-3 段目标风格的文字直接丢给 AI 读。
3. 高潮写高潮的参考，日常写日常的参考，不要混。

**这一步是硬门槛：开写正文前必须有文风样本。** 如果作者暂时没有样本，标记"文风待定"，角色构建和大纲可以先推进，但正文第一章动笔之前必须补上。没有文风锚点就开写，后面所有规则都是空中楼阁。

**如果目标风格样本和下面的规则冲突，以目标风格样本为准。** 规则是兜底，不是用来压过文风本身的。

→ 展开：`methods/writing-rules.md`

## 第一步：构建角色

AI 天然倾向标签化——给一个"温柔"，写出来就是纯温柔。角色要活，靠三层结构：

### 性格调色盘

给角色一块调色盘，不是一个词：

- **主色调**：最常见的行为模式
- **底色**：所有选择背后的底层逻辑
- **对冲/点缀**：和主色调不完全一致的那一面

最少三个，不是最多三个。每个性格必须往下衍生成具体场景和动作——"温柔"不行，"她不会直接说担心，但会默默把他杯子里的水换成热的"才行。**每个主要角色都必须有衍生，不能只停在标签层面。** 没有衍生的调色盘等于没有调色盘。

衍生不要让 AI 直出。AI 的衍生永远在"逻辑相关"范围内，只有人类才能做到"社恐的衍生一：社交达人"这种反直觉组合。衍生必须手写。

**语料：** 每个主要角色至少给出 3-5 句只有这个角色才会说的话，覆盖不同情绪状态（平常、生气、敷衍、真怕、装没事）。重点不是填满栏位，是这些话换成别的角色说不出来。

→ 模板：`templates/character-palette.template.md`

### 多面性

**每个主要角色都必须先判断需不需要多面性。** 需要就写，不需要也要写明为什么不需要——这能防止 AI 默认跳过这一步。

只有当角色在不同场景里会切换**整套生存策略**时才写——"稍微放松一点"不算，整套行为逻辑都换了才算。

需要的情况：有伪装、经历过极端创伤、在不同关系中定位完全不同。

每张面五个部件，缺一不可：

| 部件 | 回答的问题 |
|------|-----------|
| 触发条件 | 什么情况下启动？ |
| 能量状态 | 消耗多少精力？ |
| 说话方式 | 在这张面里怎么说话？ |
| 身体行为 | 怎么动？姿态、距离、下意识动作 |
| 功能 | 这张面在保护什么？ |

面与面之间是渐变过渡，不是开关切换。必须写**过渡**（怎么切换）和**渗透**（一张面运行中另一张面的泄漏）。渗透比完整切换更有味道——一秒钟的泄漏比十分钟的模式切换更暴露真实。

有多面性的角色，语料从调色盘剔除，放到每张面里。没有多面性的角色，语料留在调色盘。

"多面性"是概念名，不是数量限制。两面、三面、四面都行，由角色决定。不需要就不写——硬拆会让 AI 在模式间生硬切换。

→ 模板：`templates/multi-faces.template.md`

### 二次解释与行为禁区

二次解释 = 告诉 AI"这个性格是这个意思，不是你资料库里以为的那个意思"。用作者自己的话解释，越像私下给朋友讲"这角色其实是这样的"，效果越好。越像说明书，AI 越当模板套用。

行为禁区比正面描述有效得多。告诉 AI"她很温柔"，AI 有一万种温柔。告诉 AI"她不会直接说我担心你、不会在他人面前哭、不会主动身体接触"，行为空间立刻精确。

禁区写法：列 5-12 条角色在正常压力范围内不会做的事，标注例外条件和出现频率。**禁区描述的是常态极限，不是人格铁律。** 不要写成"绝对不会"的死命令——写成"常态下怎么表现"，给极端情况留出打破的空间。

→ 模板：`templates/behavior-boundaries.template.md`

### 防止公式化——禁区是为了被打破的

**这是整个体系中最重要的原则。**

如果角色从头到尾都严格遵守禁区，那不是稳，是死。好角色之所以打动人，恰恰是在某个极端时刻做了常态下不可能的事。

一个从不哭的人，哭了一次——这一次比哭一百次都重。

四条铁律：

1. **禁区是弓，打破是箭。** 弓拉得越满，箭射出去越远。
2. **打破必须稀有。** 全书个位数次。频繁打破 = 禁区不存在。
3. **打破后必须有恢复。** 角色不会破功了就一直软下去。她会拼命把裂缝缝回去——缝的过程本身就是最好的角色描写。
4. **情感强度匹配场景强度。** 日常用日常反应，生死允许极端。日常崩溃 = 公式化。生死嘴炮 = 也是公式化。

构建顺序：调色盘（含衍生和语料）→ 多面性判断 → 二次解释 → 行为禁区。建完用检查清单过一遍。

**顺序锁：** 角色构建和大纲可以交替推进，但**进入章级细纲之前，所有主要角色的四步必须完成。** 不能主角写满、配角敷衍——所有主要角色的四步必须达到同等深度。补充：先确认核心角色阵容（哪些人、什么关系），再逐个走四步。阵容没定就不建单个角色档案。

→ 检查清单：`templates/character-build-checklist.template.md`
→ 外貌/气质锚点：`templates/character-appearance.template.md`
→ 完整方法论：`methods/character-design.md`
→ 完整角色示例：`examples/full-character-example.md`

## 第二步：搭结构和世界观

### 大纲

全书先有总纲（主线方向、弧划分），再细化到弧级（每弧的阶段、转折、情感走向），最后到章级。

**每个主要角色在大纲中需要有自己的目标和至少一个只属于她的选择时刻。** 角色不能只在"关系轴"里作为主角的附属出现——她有自己想要的东西、自己的困境、自己必须做的决定。

**大纲以作者提供的方向为准。AI 可以提建议和补充，但不能替作者做核心决定（主线走向、角色命运、结局）。**

如果作者还没明确"最不想写成什么样"，在进入大纲之前追问一次。这条比"想写什么"更能防止滑入套路。

构思阶段不要一次抛太多问题。一次聚焦一个核心方向，作者答得模糊时再给 2-4 个选项帮助缩窄。

→ 模板：`templates/story-outline.template.md`
→ 方法论：`methods/story-structure.md`

### 世界观

不要把世界观当一整坨资料发给 AI。拆成三层：

| 层级 | 放什么 | 更新频率 |
|------|--------|---------|
| 总档 | 地图、体系、势力、底层规则 | 很少变 |
| 当前弧喂包 | 这一弧真正影响人物选择的设定 | 每弧更新 |
| 本章喂包 | 这一章当前要用的世界规则和术语 | 每章更新 |

AI 只看到你喂进去的部分。没喂的它会自己编——编出来的和你想的一定不一样。

每个新术语在它第一次出现的那一层锚定：这是什么、怎么运作、它影响谁的什么选择。AI 看到"玄气"不知道是法力还是呼吸术，你不说它就会猜，猜错了后面全错。

→ 模板：`templates/worldbuilding.template.md`
→ 方法论：`methods/worldbuilding-feed.md`

## 第三步：写作铁律

写的时候只记住三条：

1. **说出口的话和心里想的永远不是同一件事。** 角色把真实意图说清楚了，就写错了。读者感动的永远是没说出口的部分。
2. **不解释动机，只写动作。** 写"她转过身去"，不写"她因为害怕暴露柔软转过身去"。句子里出现"因为""本能""源于"来解释行为动机，删掉那个从句。
3. **叙述者要有性格。** 平稳、客观、中性的旁白 = 没有性格的叙述者 = AI 默认输出。必须打破。

写的时候不要带着禁令写——那会让 AI 拧巴。**先放开写，写完再审。**

→ 展开：`methods/writing-rules.md`

---

## 第四步：每章写之前

每章开写前准备一份输入包，只放这一章真正要用的东西：

- 本章要推进的剧情线
- 登场角色（只放当前相关的性格切面，不是全部设定）
- 世界观本章喂包
- 上一章末尾的情感温度和未完成的线索

三人以上场景，先定摄像机（这一场主要跟着谁看）、主关系轴（谁和谁的关系是这场的核心张力）和情绪节拍（谁先动、谁后动、沉默属于谁）。

→ 输入包模板：`templates/chapter-input-pack.template.md`
→ 多人场景：`methods/multi-character.md`

---

## 第五步：写完之后

先放开写，写完再审。审的时候分两步：

1. 先跑 `python tools/ban-check.py <章节文件>`，扫表层套句风险。**通过不等于没问题**，只是最表面的一层。
2. 再人工对照 `methods/review-checklist.md` 过一遍。

→ 审稿清单：`methods/review-checklist.md`
→ 反模板补丁：`patches/anti-template.md`
→ 活人感补丁：`patches/liveness-patch.md`
→ 工具说明：`tools/README.md`

---

## 第六步：长篇管理

每 3 章为一个批次，写完复盘。

长篇最容易出的问题：角色状态漂移、伏笔丢失、情感债没收。用状态表记录，每批次更新。

跨窗口接力时，不要直接贴一堆正文。先给结构化摘要和限制条件——AI 只需要知道"现在到哪了""角色目前什么状态""这一段最重要的约束是什么"。

→ 长篇管理：`methods/longform-management.md`
→ 状态表：`templates/longform-state.template.md`
→ 批次笔记：`templates/batch-notes.template.md`
→ 跨窗口接力：`templates/handoff.template.md`
→ 快照：`templates/session-snapshot.template.md`

## 项目边界

- 这不是全自动写书引擎。
- 这不是现成的多 Agent 生产线。
- 这是一套帮助你把小说写得更稳、更像人、更不容易模板化的公开框架。
