# 活人感补丁

这份补丁的目标不是加新设定，而是防止人物只剩剧情功能。

## 先记住

1. **人物先是人，再是剧情工具。**
   先写他此刻在做什么，再写这一章要推进什么。

2. **每章至少给主要角色一个“无硬剧情功能”的小动作。**
   可以是停一下、洗手慢半拍、看向别处、把话咽回去、站着不走、伸手又收回。

3. **少写结论句、解释句、分析句。**
   少写“他意识到”“她其实是”“这说明”“逻辑上”“策略上”。
   优先写动作、停顿、视线、语气、答非所问。

4. **允许人物不最优。**
   活人会绕一下、拖一下、嘴硬一下、逞一下、心软一下。
   不要让人物永远做最正确、最高效、最利于剧情的反应。

5. **对话不要总是正中靶心。**
   角色可以先敷衍、先岔开、先回一句别的，再慢慢露真东西。

## 常见修正方向

### 当角色像功能件时

把这类句子收回来：

- “他决定采用更有效率的方式”
- “她从逻辑上判断”
- “这说明他已经成熟了”

改成：

- 一个动作
- 一个卡住的回答
- 一个说到一半断掉的句子
- 一个和嘴上说法不一致的身体反应

### 当情绪太“解释”时

不要急着写：

- 他很难过
- 她其实很在意
- 他意识到自己害怕

先问：

- 他难过时会先做什么？
- 她在意时会多看一眼，还是回话变慢？
- 他害怕时会变安静，还是变得话更多？

## 三个快速检查

1. 这一章里，有没有一个“只有这个角色才会做”的动作？
2. 有没有一句本来想解释，最后改成了动作或停顿？
3. 这一章里，人物有没有一次“不最优但很像他”的反应？

## 角色写崩的常见信号

- 只会按设定稳定运行
- 一到关键处就开始像解题
- 每次都说最准确的话
- 每次都做最利于推进的事
- 感情变化全靠结论句交代

一句话提醒：

**活人感不是“设定稳定运行”，而是“这个人今天也会像他自己一样活一下”。**
