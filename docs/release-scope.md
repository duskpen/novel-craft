# 公开范围说明

这个仓库是从一个正在写作中的私有项目里抽出来的公开版。

目标不是公开某一部小说，而是公开一套可复用的创作方法、补丁和模板。

## 公开原则

1. 方法论优先，项目素材后退。
2. 公开骨架，不公开正在使用的正文和设定答案。
3. 保留工作流，删掉会泄露具体项目的信息。
4. 示例只做脱敏小样，不做真实项目镜像。

## 可以公开的内容

- 通用写作方法论
- 审稿清单
- 结构设计方法
- 长篇状态管理方法
- 反模板补丁
- 活人感补丁
- 可直接复制的空白模板
- 脱敏后的极小示例

## 默认不公开的内容

- 正在创作中的角色专档
- 专案世界观总档
- 全书总纲、分弧大纲
- 已写章节正文
- 批次真实进度
- 正在使用的接力提示词
- 会暴露后续剧情的隐性共识

## 从私有文件到公开文件的转换规则

| 私有文件类型 | 公开处理方式 |
| --- | --- |
| 角色档案 | 改成空白模板或小型通用示例 |
| 世界观/大纲 | 不公开；只提炼方法，不公开答案 |
| `HANDOFF.md` | 提炼成 `templates/handoff.template.md` |
| `SESSION-SNAPSHOT.md` | 提炼成 `templates/session-snapshot.template.md` |
| `novel/batch-notes.md` | 提炼成 `templates/batch-notes.template.md` |
| 最近章节 | 不公开；只在 README 里说明推荐读取“最近三章”这类工作法 |

## 脱敏时必须做的事

- 删除专名、地名、门派名、家族名
- 删除当前章号、当前进度、下一批任务
- 删除未公开剧情答案和伏笔去向
- 删除真实角色关系温度与隐藏信息
- 把“这本书是怎么做的”改写成“这类项目可以怎么做”

## 发布前自查

发布任何新文档前，先问四个问题：

1. 陌生人能不能从这份文档反推出原项目的剧情或设定？
2. 这份内容是不是仍然依赖原项目的专名和上下文？
3. 它更像“案例答案”，还是“可复用模板”？
4. 别人复制这份文件后，能不能直接用到自己的小说里？

只要前两题有一题是“能”，就继续抽象。
只要后两题有一题是“不能”，就继续重写。
