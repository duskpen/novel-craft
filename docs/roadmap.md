# 路线图

`novel-craft` 的第一阶段不是去伪装成自动化写作平台，而是先把最值钱的质量控制内核公开出来。

## Phase 1：公开框架仓库

当前阶段目标：

- 公开方法论文档
- 公开反模板与活人感补丁
- 公开可复制模板
- 给出推荐工作流
- 明确项目边界

这一阶段的关键词是：`文档化`、`模板化`、`去项目专名化`。

## Phase 2：守门员工具

先做不代笔、只防跑偏的工具。

优先级建议：

1. 禁句式检查
2. 反模板检查
3. 段落节奏提醒
4. 批次审稿辅助

这一阶段的目标不是写得更多，而是写得更稳。

## Phase 3：状态工具

在守门员工具稳定之后，再做状态层工具。

优先级建议：

1. 本章输入包生成器
2. 批次记录整理器
3. 接力包导出器
4. 角色状态 / 情感债 / 未收线索的结构化存储

这一层的核心价值是：减少换窗口和长篇推进时的失真。

## Phase 4：半自动流水线

到这一阶段，才值得考虑：

- writer
- auditor
- reviser
- snapshot
- rollback

但这里有一个前提：

**工具层不能反过来伤害质量层。**

如果自动写章让角色越来越像功能件，那就说明流水线跑得太快了。

## 当前非目标

以下内容当前不作为首发目标：

- 一键生成整本小说
- 全自动多 Agent 写作
- 平台级在线产品
- 把作者完全移出创作环节

## 方向判断

`novel-craft` 更适合走这条路：

1. 先把“角色活人感 + 反模板 + 长篇不失真”做成公开标准件
2. 再把这些标准件工具化
3. 最后才考虑自动化写作外壳

一句话总结：

**先做质量内核，再做工程外壳。**
