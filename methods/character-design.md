# 角色构建

这份文档解决的不是“角色设定够不够多”，而是“角色为什么总像一个标签在运行”。

对 AI 协作写作来说，角色真正容易崩的地方通常有三个：

- 只有标签，没有衍生
- 只有常态，没有切换
- 只有设定，没有禁区

## 一、性格调色盘

不要只给角色一个词。

比起“温柔”“冷淡”“活泼”，更稳的做法是给角色一块调色盘：

- `主色调`：最常见的行为模式
- `底色`：所有选择背后的底层逻辑
- `对冲/点缀`：和主色调不完全一致的那一面

### 最小写法

```markdown
角色：
主色调：
底色：
对冲/点缀：
```

### 关键点

1. 最少三个，不是最多三个。
2. 不要停在抽象词，必须继续往下衍生。
3. 衍生要落到动作、说话方式、压力反应和关系选择。

### 示例

```markdown
角色：林砚
主色调：圆滑
底色：怕失去来之不易的位置
对冲/点缀：偶尔会硬一下，不肯退

衍生：
- 他平时什么话都接得住，但一旦碰到“替别人背锅”，说话会突然变硬。
- 他会先看场面、看人脸色，再决定怎么开口。
- 真被逼到退无可退时，他不一定最聪明，但会突然不让。
```

## 二、多面性

不是每个角色都需要多面性。

只有当角色在不同场景里会切换整套生存策略时，才值得写。

### 什么时候需要

- 有明显伪装
- 对外和对内是两套人
- 在安全关系和高压关系里反应完全不同

### 每张面的最小字段

```markdown
这张面的名字：
触发条件：
能量状态：
说话方式：
身体行为：
这张面的功能：
```

### 别漏掉的两件事

1. `过渡`：角色不是按开关切面。
2. `渗透`：一张面运行时，另一张面的东西会漏出来一点。

真正有味道的往往不是完整切换，而是漏出来的那半秒。

## 三、二次解释

二次解释的作用，是防止 AI 把一个标签往它熟悉的方向套。

例如你写“高傲”，AI 很可能自动写成“刻薄 + 嘲讽 + 冷笑”。
但你想要的也许是“习惯掌控局面，不爱解释，不轻易求人”。

所以你要把“你真正想要的版本”说明白。

### 最小写法

```markdown
这个标签在本书里是什么意思：
它常出现在哪些场景：
它不等于什么：
AI 最容易误写成什么：
```

## 四、行为禁区

行为禁区比“她很温柔”“他很冷”更有用。

因为禁区会直接缩小 AI 的表演空间。

### 最小写法

```markdown
角色：
常态下绝不会做的事：
- 
- 
- 

例外条件：

例外出现频率：
```

### 写禁区时要记住

1. 禁区不是永远不能破。
2. 禁区是为了在真正重要的时候被打破。
3. 打破之后，角色通常会试图把裂口补回去。

如果一个角色从头到尾都稳定运行、不曾偏离，那不是稳，是死。

## 五、构建顺序

推荐顺序：

1. 先写调色盘
2. 再写多面性
3. 再写二次解释
4. 最后写行为禁区

一句话总结：

**调色盘回答“他是谁”，多面性回答“他怎么运作”，行为禁区回答“他什么时候最伤人也最动人”。**
