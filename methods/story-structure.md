# 剧情结构与章节设计

角色写活了，不等于章节就成立了。

这份文档关心的是：这一章为什么有劲，为什么没劲，为什么明明句子不差却读不下去。

## 一、结构真正管理的东西

结构不是打卡式地塞“冲突、高潮、反转”。

结构真正管理的是两件事：

- 读者此刻在期待什么
- 这一章结束后，期待有没有被推进、兑现、推迟或扭转

一句话判断一章是否成立：

**如果这一章结束后，人物关系、信息状态、情绪温度、局势压力四者里没有任何一项变化，这章大概率可以删、并、或重写。**

## 二、三层结构

### 1. 场景

场景尽量具备四件套：

- `目标`：角色此刻想要什么
- `阻力`：谁或什么在拦他
- `转折`：场景中途发生了什么变化
- `残留`：结束后还留了什么没完

没有残留，场景很容易像一扇关死的门。

### 2. 章节

一章最好具备这四个部件：

- `开头锚点`：把读者拉进当前状态
- `中段升级`：至少一条轴往上走
- `变化落点`：这一章真正发生了什么变化
- `章尾残留`：留下一点没说完、没做完、没消化完的东西

### 3. 弧

一弧的作用不是凑很多章，而是：

- 提出问题
- 逐步加压
- 逼角色做选择
- 让选择带来后果

## 三、场景卡

写场景前，先写一张小卡：

```markdown
场景目的：
当前视角：
角色此刻想要：
当前阻力：
场景中的变化：
场景结束后的残留：
```

### 什么算变化

以下都算变化：

- 得到一个新信息
- 误会更深了
- 情绪裂开一点
- 关系前进半步
- 一件本来还能拖的事，现在拖不下去了

## 四、章节卡

开写前先回答：

**如果删掉这一章，整部小说会失去什么？**

答不上来，就说明这一章存在风险。

推荐用这张卡：

```markdown
本章存在理由：
本章主轴：
本章起点状态：
本章要推进的两个轴：
本章中段升级：
本章变化落点：
章尾残留：
为后文埋下：
```

“两个轴”通常从这里选：

- 剧情
- 关系
- 人物自我认知
- 外部局势

## 五、重场景别写崩

重逢、决裂、告白、牺牲、摊牌、和解，这些都不是“写响一点”就成立。

真正有重量的重场景，前面最好至少铺到三层：

1. `信息铺垫`：读者知道这件事迟早会来
2. `情绪铺垫`：角色已经绕着它活了很久
3. `阻力铺垫`：读者知道它发生要付代价
4. `记忆铺垫`：前面已有反复出现的动作、话、意象、承诺

### 写之前先问

1. 读者知道它为什么重要吗？
2. 角色知道自己可能失去什么吗？
3. 前面至少预约过两次吗？
4. 发生之后，会带来明确后果吗？

只要有两条答不上来，就先补铺垫，不要急着把大场面写满。

## 六、桥梁章

很多书不是高潮写崩，而是高潮和高潮之间全是空地。

桥梁章的任务不是“过渡一下”，而是做这些事：

- 收前一章的余波
- 为下一次抬升存债
- 调整关系温度
- 让世界状态继续往前走

桥梁章不是废戏。桥梁章是张力的续命器。

## 七、收尾提醒

不是每章都需要强钩子，但每批章节里至少要有一次明显的节奏外拉：

- 风险浮出
- 信息翻面
- 关系偏移
- 小冲突抬头
- 下一步被逼出来

一句话总结：

**结构不是让每章更响，而是让每章结束时，读者都能感觉到“东西动了”。**
