# 长篇管理

长篇最容易死掉的地方，通常不是某一句不够好，而是：

- 角色慢慢不对位
- 情绪债断掉
- 伏笔蒸发
- 时间线打架
- AI 只记得最近最响的东西

这份文档的核心观点很简单：

**长篇不是靠 AI 自然记住，而是靠外部状态管理。**

## 一、三层管理系统

把长篇拆成三层：

| 层级 | 放什么 | 更新频率 |
| --- | --- | --- |
| 静态设定层 | 角色底层设定、世界观、文风方向 | 很少变 |
| 动态状态层 | 当前关系、情绪、未收线索、时间线 | 高频更新 |
| 本章输入层 | 这一章真正要喂给 AI 的最小信息包 | 每章更新 |

### 原则

1. 不要一次喂完所有资料。
2. 动态状态优先于百科式设定。
3. 回收管理比记忆堆积更重要。

## 二、动态状态层

### 1. 角色状态表

每个核心角色至少记录：

```markdown
角色：
当前最想要：
当前最怕失去：
当前知道什么：
当前不知道什么：
当前在藏什么：
和关键角色的关系温度：
当前身体/精神状态：
```

注意写“现在”，不是角色百科。

### 2. 情感债账本

很多长篇后期发虚，是因为角色之间欠下的东西没人记。

情感债可以是：

- 一句没说出口的话
- 一次没完成的回应
- 一次该来的道歉
- 一次迟早要爆的情绪
- 一次保护、误会、兑现、告别

```markdown
欠债人：
债主：
债的类型：
债是怎么欠下的：
为什么现在还没还：
适合在哪类场景里回收：
紧急程度：低 / 中 / 高
```

### 3. 未收线索表

```markdown
线索：
第一次出现在哪里：
它向读者承诺了什么：
短期提醒节点：
预计回收窗口：
当前状态：未动 / 提醒过 / 半回收 / 已回收
```

重点不是“记住有这条线”，而是“记住你已经向读者承诺了什么”。

### 4. 时间线与世界状态

最少记录：

```markdown
章号：
日期/相对时间：
地点：
发生的大事：
身体状态变化：
世界状态变化：
```

## 三、本章输入包

每章开写前，不要喂全书百科，只喂当前真正相关的东西。

推荐模板：

```markdown
章号：
视角：
时间地点：
上一章残留：
本章存在理由：
本章主轴：
本章要推进的两个轴：

相关角色状态（最多 3-4 人）：
- 

本章相关情感债（最多 2-3 条）：
- 

本章相关未收线索（最多 2-3 条）：
- 

绝对不能忘的事实（最多 5 条）：
- 

目标风格样本：
```

### 输入原则

1. 小而准
2. 动态优先
3. 状态优先于百科
4. 风格样本一定要带

## 四、批次工作流

推荐按 `3 章` 为一个批次推进。

每批要做的事：

1. 写完正文
2. 跑审稿清单
3. 记录这一批推进了什么
4. 更新角色状态、情感债、未收线索、时间线
5. 复盘哪些手法已经开始高频，下一批要避开什么

## 五、跨窗口接力

如果你经常换 AI 窗口，最容易损耗的不是设定本身，而是：

- 关系温度
- 隐性边界
- 已经用过的互动模板
- 这一阶段到底该写得多热、多冷、多慢、多快

所以接力时不要只贴设定。

至少要交出这几类信息：

- 当前真实进度
- 角色当前状态
- 最近一批的后果
- 下一批的任务边界
- 不能复用的模板
- 这一阶段的语感提醒

## 六、复盘节奏

建议每 `3-5 章` 小复盘一次，每一弧大复盘一次。

小复盘看：

- 最近三章是不是太像
- 有没有哪个角色开始只剩一种反应
- 有没有线索超过预期还没被提醒
- 有没有情绪债该收不收

大复盘看：

- 这条弧提出的问题有没有兑现
- 谁被逼到了边缘
- 哪段关系真的变了
- 高潮后的余波够不够

一句话总结：

**长篇管理不是把所有东西都记下来，而是只抓那些将来必须回来、必须变重、必须算账的东西。**
