# 接力模板

把下面这段改成你自己的项目路径和文件名，再发给新窗口 AI。

```markdown
我在续写一个小说项目，所有资料都在 `<你的项目路径>`。请不要直接开写，先按顺序读取下面这些文件：

1. `<项目>/SKILL.md`
2. `<项目>/methods/review-checklist.md`
3. `<项目>/methods/story-structure.md`
4. `<项目>/methods/longform-management.md`
5. `<项目>/patches/anti-template.md`
6. `<项目>/patches/liveness-patch.md`
7. `<项目>/当前批次记录.md`
8. `<项目>/当前窗口隐性共识快照.md`
9. `<项目>/最近三章`

补充要求：

1. 实际已写正文 > 最新批次记录 > 隐性共识快照 > 旧大纲和旧任务书。
2. 如果有历史任务书，只看它怎么锁边界，不要机械按旧章号继续写。
3. 读完后，先总结：
   - 当前人物状态
   - 当前关系温度
   - 上一批留下了什么后果
   - 下一批任务边界
   - 哪些模板不能复用
   - 这一阶段人物该怎么写得更像活人
4. 等我确认后再继续写。

用简体中文回复。
```

## 使用提醒

- 接力时不要只丢设定档。
- 一定要把“最近三章 + 批次记录 + 隐性共识快照”一起交出去。
- 新窗口最容易丢的不是设定，而是关系温度、语感边界和已经用过的模板。
