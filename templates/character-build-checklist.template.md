# 角色构建检查清单

写完角色设定后，逐项检查。

```markdown
## 调色盘

- [ ] 有主色调、底色、对冲/点缀（至少三个）
- [ ] 每个性格有至少 2-3 条衍生
- [ ] 衍生里有反直觉组合（不全是逻辑相关的）
- [ ] 衍生是具体场景而非抽象描述

## 多面性（如果需要）

- [ ] 每张面有完整五部件（触发 / 能量 / 语料 / 身体 / 功能）
- [ ] 语料从调色盘移到了多面性的每张面里
- [ ] 每张面的压力性质确实不同（否则合并）
- [ ] 写了至少两组过渡（最常见 + 最剧烈的切换）
- [ ] 写了渗透（一张面运行中其他面的泄漏）

## 二次解释

- [ ] 针对每个关键性格点有作者理解
- [ ] 防止了 AI 最可能的误读方向
- [ ] 行为禁区列了 5-12 条"不会做的事"
- [ ] 禁区标注了例外条件和出现频率
- [ ] 禁区不是铁律——标注了"什么极端情况下可以打破"和"打破后怎么恢复"

## 防止公式化

- [ ] 角色不是从头到尾都严格遵守禁区——极端场景有超出常态的反应
- [ ] 打破禁区后有恢复动作（角色会试图把裂缝缝回去）
- [ ] 情感强度匹配场景强度（日常不崩溃，生死不嘴炮）

## 写完后审稿

- [ ] 角色说出口的话和心里想的不是同一件事
- [ ] 叙述者有性格，不是中性旁白
- [ ] 跑一遍 `methods/review-checklist.md` 的 AI 坏习惯清单
- [ ] 跑一遍 `tools/ban-check.py`
```
