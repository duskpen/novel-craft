# 工具说明

当前 `tools/` 目录只放低风险、守门员型工具。

这些工具的目标不是代替写作，而是：

- 在写完之后帮你抓明显套句
- 把人工复查里最机械的一层先自动化
- 让公开仓库不止有文档，也有最小可用的可执行工具

## ban-check.py

扫描 `.md` / `.txt` 文本里的常见风险信号，包括：

- “不是X，是Y”句式
- 直接心理描写
- 沉默后补一句解释
- 解释性动机从句
- 过于直白的情绪点名
- 单行比喻过密

### 用法

```bash
python tools/ban-check.py novel/chapter-051.md
python tools/ban-check.py drafts/
```

### 退出码

- `0`：未命中当前规则覆盖的表层风险，**不代表这章没有问题**
- `1`：发现问题
- `2`：参数错误或路径不存在

`ban-check.py` 只扫最表面的套句风险。通过检查不等于质量过关，人工审稿清单（`methods/review-checklist.md`）仍然是必须的。
